# find使用手册

[toc]

find - 递归地在层次目录中处理文件，用法如下

```bash
find [path...] [expression：表达式]
path：路径
expression：表达式
```

## 描述

`find` 会遍历目录树内所有文件，**从左至右**解析给定的表达式，并遵循运算符优先级规则进行匹配，直到得出结果，然后 find 移向下一个文件名。

* 如果遇到 **"与"（and）** 条件（比如 `-name "*.txt" -size +1M`）：左边是假，右边就不用看了（因为已经是假了），直接判定这个文件不符合条件
* 如果遇到 **"或"（or）** 条件（比如 `-name "*.txt" -o -name "*.md"`）：左边是真，右边也不用看了（因为已经符合了），直接判定这个文件符合条件

第一个以 '-' , '(' , ')' , ',' 或 '!' 这些字符起始的参数是表达式的开始；在它之前的任何参数是要搜索的路径，在它之后的任何参数都是表达式的余下部分。

* 从左往右看参数，一旦遇到以 `-`、`(`、`)`、`,`、`!` 开头的参数，就认为**从这里开始就是表达式了**，这个参数之前的所有参数都是要搜索的路径

```bash
find [路径] [表达式]

# 查找所有不是 .txt 结尾的文件
find . ! -name "*.txt" (！前是是路径，后面是表达式剩余的内容)

# 或者写成（加空格和转义，避免 shell 误解）
find . \! -name "*.txt"

# 找 .txt 或 .md 文件，同时文件名包含 "readme"
find . \( -name "*.txt" -o -name "*.md" \) -name "*readme*"
```

当所有文件都成功处理时 find 退出并返回状态值0。如果发生错误则返回一个大于0的值。

## 表达式

表达式由**选项**、**测试**、**动作**组成，彼此通过运算符分隔：

1. **选项**：作用于所有文件，始终返回真值
2. **测试**：对文件做条件判断，返回真 / 假
3. **动作**：产生额外操作效果，同时返回真 / 假

当未显示书写运算符时，默认使用 -and（逻辑与）连接，若表达式中不含 `-prune` 以外的动作，表达式结果为真时自动执行 `-print`

```bash
find . -name "*.txt"
# -name "*.txt" 是一个测试，没有选项也没有动作，结果为真时自动补上 -print

find . -name "*.txt" -mtime -7
# -name "*.txt" → 测试 1， -mtime -7 → 测试 2（7 天内修改）。没有写运算符 → 自动用 -and 连接

find . -maxdepth 2 -name "*.log" -ls
# -maxdepth 2 → 选项（永远为真），-name "*.log" → 测试，-ls → 动作，没有运算符 → 全部 -and 连接

find . -path "./tmp" -prune -o -name "*.txt"
# 有 -prune，不会自动加 -print
```

### 选项（Options）

用来改变`find`的工作方式，通常放在表达式最前面，总是返回“真”，影响全局行为。

| 选项                    | 说明                                                         |
| :---------------------- | :----------------------------------------------------------- |
| `-daystart`             | 从当日起始时开始而不是从 24 小时之前，计算时间（用于 -amin, -atime, -cmin, -ctime, -mmin, -mtime） |
| `-depth`                | 先处理目录的内容再处理目录本身                               |
| `-follow`               | 不检索符号链接。隐含了 -noleaf                               |
| `-help`, `--help`       | 列出 find 的命令行用法的概要，然后退出                       |
| `-maxdepth levels`      | 进入命令行参数指定的目录下层目录时，最深不超过 levels 层。`-maxdepth 0` 意味着只在命令行参数指定的目录中执行测试和动作 |
| `-mindepth levels`      | 不在 levels 层之内执行任何测试和动作。`-mindepth 1` 意味着处理所有的文件，除了命令行参数指定的目录中的文件 |
| `-mount`                | 不进入处于其它文件系统之上的目录。可以用 -xdev 代替          |
| `-noleaf`               | 不为"目录中子目录数量比硬连接数少 2"这种假设做优化。用于 CD-ROM、MS-DOS、AFS 等特殊文件系统 |
| `-version`, `--version` | 打印 find 的版本号然后退出                                   |
| `-xdev`                 | 不进入处于其他文件系统之上的目录                             |

```bash
find . -maxdepth 2 -name "*.txt"
#      ↑
#    选项：只搜索2层深度
```

### 测试（Tests）

用于检查文件是否满足条件，不改变文件。

针对时间、大小、数量等数值判断，统一规则：

- `+n`：大于 n
- `-n`：小于 n
- `n`：恰好等于 n

| 测试              | 说明                                                         |
| :---------------- | :----------------------------------------------------------- |
| `-amin n`         | 对文件的最近一次访问是在 n 分钟之前                          |
| `-anewer file`    | 对文件的最近一次访问比 file 修改时间要晚                     |
| `-atime n`        | 对文件的最近一次访问是在 n×24 小时之前                       |
| `-cmin n`         | 对文件状态的最近一次修改是在 n 分钟之前                      |
| `-cnewer file`    | 对文件状态的最近一次修改比 file 修改时间要晚                 |
| `-ctime n`        | 对文件状态的最近一次修改是在 n×24 小时之前                   |
| `-empty`          | 文件是空的普通文件或者空目录                                 |
| `-false`          | 总是 false                                                   |
| `-fstype type`    | 文件处于 type 类型的文件系统之上（如 ufs, nfs, tmp 等）      |
| `-gid n`          | 文件的数字形式的组 ID 是 n                                   |
| `-group gname`    | 文件属于 gname（也允许使用数字形式的组 ID）                  |
| `-ilname pattern` | 和 -lname 类似，但匹配时**不区分大小写**                     |
| `-iname pattern`  | 和 -name 类似，但匹配时**不区分大小写**                      |
| `-inum n`         | 文件的 i 结点数是 n                                          |
| `-ipath pattern`  | 和 -path 类似，但匹配时**不区分大小写**                      |
| `-iregex pattern` | 和 -regex 类似，但匹配时**不区分大小写**                     |
| `-links n`        | 文件有 n 个链接                                              |
| `-lname pattern`  | 文件是一个与 pattern 匹配的符号链接                          |
| `-mmin n`         | 对文件数据的最近一次修改是在 n 分钟之前                      |
| `-mtime n`        | 对文件数据的最近一次修改是在 n×24 小时之前                   |
| `-name pattern`   | 基本文件名与 shell 模式 pattern 相匹配                       |
| `-newer file`     | 对文件的最近一次修改比 file 修改时间要晚                     |
| `-nogroup`        | 没有符合文件的数字形式的组 ID 的组                           |
| `-nouser`         | 没有符合文件的数字形式的用户 ID 的用户                       |
| `-path pattern`   | 文件名与 shell 模式 pattern 相匹配                           |
| `-perm mode`      | 文件的权限位恰好是 mode                                      |
| `-perm -mode`     | 所有权限位 mode 都被设置了的文件                             |
| `-perm +mode`     | 任何权限位 mode 被设置了的文件                               |
| `-regex pattern`  | 文件名与正则表达式 pattern 匹配（对整个路径的匹配）          |
| `-size n[bckw]`   | 文件使用了 n 单位个存储单元（默认 512 字节块，`c`=字节，`k`=KB，`w`=2字节字） |
| `-true`           | 总是 true                                                    |
| `-type c`         | 文件是 c 类型的（`b`块设备、`c`字符设备、`d`目录、`p`管道、`f`普通文件、`l`符号链接、`s`套接字、`D`门） |
| `-uid n`          | 文件的数字形式的用户 ID 是 n                                 |
| `-used n`         | 文件最后一次存取是在最后一次修改状态的 n 天之后              |
| `-user uname`     | 文件的所有者是 uname（也允许数字形式的用户 ID）              |
| `-xtype c`        | 和 -type 相同，除非文件是一个符号链接（对符号链接有特殊处理） |

```bash
find . -name "*.txt" -size +1M
#      ↑               ↑
#    测试1           测试2(大小大于1MB？)
```



### 动作 ACTIONS

对文件执行操作

| 动作                   | 说明                                                         |
| :--------------------- | :----------------------------------------------------------- |
| `-exec command ;`      | 执行 command；如果命令返回状态值 0，那么 exec 返回 true。`{}` 将被正在处理的文件名替换 |
| `-fls file`            | 返回 true；类似 -ls 但写入 file                              |
| `-fprint file`         | 返回 true；将文件全名打印到文件 file 中                      |
| `-fprint0 file`        | 返回 true；类似 -print0 但写入 file                          |
| `-fprintf file format` | 返回 true；类似 -printf 但写入 file                          |
| `-ok command ;`        | 类似 -exec 但会先向用户询问确认                              |
| `-print`               | 返回 true；在标准输出打印文件全名，然后是一个换行符          |
| `-print0`              | 返回 true；在标准输出打印文件全名，然后是一个 null 字符      |
| `-printf format`       | 返回 true；按 format 格式打印（支持 `\` 转义和 `%` 指令）    |
| `-prune`               | 如果没有给出 -depth 则返回 true；不进入当前目录              |
| `-ls`                  | 返回 true；以 `ls -dils` 格式列出文件                        |

```bash
find . -name "*.tmp" -delete
#                     ↑
#                   动作：删除文件
```

#### -printf 格式说明

**常用转义序列：**

| 转义 | 说明         |
| :--- | :----------- |
| `\n` | 新行         |
| `\t` | 水平 tab     |
| `\r` | 回车         |
| `\\` | 输出自身 `\` |

**常用 % 指令：**

| :--- | :------------------------------------------- |
| `%p` | 文件名                                       |

| `%h` | 文件名的前面的目录部分                       |
| `%s` | 文件大小，以字节为单位                       |
| `%k` | 文件大小，以 1kB 的块为单位                  |
| `%b` | 文件大小，以 512 字节的块为单位              |
| `%m` | 文件的权限位（8 进制）                       |
| `%n` | 文件的硬连接数                               |
| `%i` | 文件的 i 结点号                              |
| `%u` | 文件的用户名                                 |
| `%U` | 文件的数字形式的用户 ID                      |
| `%g` | 文件的组名                                   |
| `%G` | 文件的数字形式的组 ID                        |
| `%a` | 文件最后一次存取的时间                       |
| `%t` | 文件最后一次修改的时间                       |
| `%c` | 文件状态最后一次修改的时间                   |
| `%d` | 文件在目录树中的深度                         |
| `%F` | 文件所在文件系统的类型                       |
| `%l` | 符号链接的目标                               |

### 运算符

以优先级高低顺序排列：

| 运算符                                                  | 说明                                                         |
| :------------------------------------------------------ | :----------------------------------------------------------- |
| `( expr )`                                              | 强制为优先                                                   |
| `! expr` 或 `-not expr`                                 | 如果 expr 是 false 则返回 true                               |
| `expr1 expr2` 或 `expr1 -a expr2` 或 `expr1 -and expr2` | **与**（默认隐含运算符）；如果 expr1 为 false 则不会执行 expr2 |
| `expr1 -o expr2` 或 `expr1 -or expr2`                   | **或**；如果 expr1 为 true 则不会执行 expr2                  |
| `expr1 , expr2`                                         | **列表**；两者都执行，列表的值是 expr2 的值                  |

## 综合示例

```bash
find /var/log -name "*.log" -mtime +30 -delete
#      ↑         ↑            ↑         ↑
#    路径      测试1        测试2     动作
#
# 执行过程：
# 1. -name "*.log"  → 是.log文件吗？
# 2. -mtime +30     → 30天前修改的吗？
# 3. -delete        → 两个条件都满足，删除文件
#
# 因为有 -delete 动作，所以不会自动打印

find . -type f -size 0
#      ↑        ↑
#    测试1    测试2
#
# 执行过程：
# 1. 没有指定动作 → 触发默认规则
# 2. 当测试全部为真时 → 自动执行 -print
# 3. 打印所有0字节的普通文件

# 命令1：会打印所有 .txt 文件
find . -name "*.txt"

# 命令2：不会打印，只会删除
find . -name "*.txt" -delete

# 命令3：会先删除，再打印（但文件已删除，可能看不到效果）
find . -name "*.txt" -delete -print

# 命令4：使用 -ok 需要确认
find . -name "*.txt" -ok rm {} \;
# -ok 是动作，所以不会自动 -print
```
